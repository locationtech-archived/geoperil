Description
    The status API accepts HTTP POST requests and returns status information about the computation of a specific event.
    URL
    http://localhost/srv/status
    Input parameter
    Property Description Status
    apikey A string of 32 characters that authorizes the usage of this API. It is linked to a user or institution account. Mandatory
    evtid The unique ID of the event for which information should be retrieved. The latest refinement will be returned, if this is a GEOFON identifier. Mandatory
    Return value
    Property Description Status
    status States the status of the request. Can be one of the following:
    • success: The operation was successfully accomplished. The values of the remaining fields are valid.
    • failure: An error occurred while trying to apply the request. This may be caused by invalid input parameter values.
    • denied: The access to this API was denied. The given API key is incorrect or not valid in conjunction with the API used.
    		Always valid.
    comp States the status of the computation. Can be one of the following:
    • success: The computation was successfully started and is completed.
    • pending: The computation was successfully started and is running.
    • progress: The computation is in the queue while waiting to be started.
    • failed: The computation was started but failed and simulation results are not available.
    • none: The computation was started but failed and simulation results are not available.
    		Always valid.
    progress Contains a number indicating the percentage already completed by the simulation computation. Always valid.
    msg Contains further information if feasible. Valid, e.g. if comp is set to failed or status is set to failure.
    API Examples
    $ curl --data "apikey=***&evtid=tcapidoc0001" http://localhost/srv/status
